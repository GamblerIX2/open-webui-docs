## 概览

本指南说明如何在 Azure Kubernetes Service（AKS）上配置 Open WebUI，并使用 Workload Identity authentication 访问 Azure OpenAI。

Workload Identity 允许 AKS pods 通过 Azure Entra ID（原 Azure AD）访问 Azure 服务，而无需在集群中存储任何凭证。这是一种安全、托管式的身份方案，可用于访问 Azure OpenAI。

## 前提条件

### OpenWebUI

- **Open WebUI**：版本为 0.6.30 或更高
- **AKS 集群配置**：你的 AKS 集群必须启用以下能力
  - **OIDC Issuer**
    - 参考 [Microsoft Learn - Create an OpenID Connect provider on Azure Kubernetes Service (AKS)](https://learn.microsoft.com/en-us/azure/aks/use-oidc-issuer)
    - AzureRM Terraform: [`azurerm_kubernetes_cluster.oidc_issuer_enabled`](https://registry.terraform.io/providers/hashicorp/azurerm/latest/docs/resources/kubernetes_cluster#oidc_issuer_enabled-1)
  - **Workload Identity**
    - 参考 [Microsoft Learn - Use Microsoft Entra Workload ID with Azure Kubernetes Service (AKS)](https://learn.microsoft.com/en-us/azure/aks/workload-identity-overview?tabs=dotnet)
    - AzureRM Terraform: [`azurerm_kubernetes_cluster.workload_identity_enabled`](https://registry.terraform.io/providers/hashicorp/azurerm/latest/docs/resources/kubernetes_cluster#workload_identity_enabled-1)

## Terraform 配置

下面是一份完整的 Terraform 配置示例，用于配置 Workload Identity authentication：

### 1. 创建 Kubernetes Namespace

首先需要创建一个 Kubernetes namespace，这样后续就能把它统一用于 Helm 部署，也能在后续步骤中赋给 user-assigned identity 和 federated identity credential。

```hcl
resource "kubernetes_namespace" "this" {
  metadata {
    name = var.kubernetes_namespace
  }
}
```

### 2. 创建 User Assigned Identity

首先创建一个 Azure User Assigned Identity，后续 Open WebUI pods 将使用它来访问 Azure 服务。

```hcl
resource "azurerm_user_assigned_identity" "uai" {
  location            = var.location
  name                = var.workload_identity_name
  resource_group_name = var.resource_group_name
}
```

### 3. 创建 Federated Identity Credential

Federated Identity Credential 会在 Kubernetes service account 与 Azure User Assigned Identity 之间建立信任关系，从而允许 pods 使用 Kubernetes token 交换 Azure token。

```hcl
resource "azurerm_federated_identity_credential" "federated_identity" {
  name                = "federated-identity"
  resource_group_name = var.resource_group_name
  audience            = ["api://AzureADTokenExchange"]
  issuer              = data.terraform_remote_state.aks.outputs.oidc_issuer_url
  parent_id           = azurerm_user_assigned_identity.uai.id
  subject             = "system:serviceaccount:${kubernetes_namespace.this.metadata[0].name}:${var.kubernetes_service_account_name}"
}
```

### 4. 为 Azure OpenAI 访问分配 RBAC 角色

在 Azure 与 User Assigned Identity 建立信任关系后，你就可以为这个 identity 分配角色。这里我们赋予它 `Cognitive Services OpenAI User` 角色；如果将来你还需要访问其他 Azure 能力，也可以继续添加更多 role assignments。

:::note
默认情况下，User Assigned Identity 对任何资源都没有访问权限，因此你必须显式授予所需 Azure RBAC 角色。
:::

:::warning
请务必将 `YOUR_COGNITIVE_ACCOUNT_ID` 替换为你 Azure OpenAI 实例对应的 cognitive account id。
:::

```hcl
resource "azurerm_role_assignment" "workload_identity_azure_openai" {
  scope                = "YOUR_COGNITIVE_ACCOUNT_ID"
  role_definition_name = "Cognitive Services OpenAI User"
  principal_id         = azurerm_user_assigned_identity.uai.principal_id
}
```

### 5. 通过 Helm 部署 Open WebUI

下面会把 Open WebUI 部署到 AKS 集群中，并配置好所需的 service account annotations 与 pod labels，以启用 Workload Identity authentication。

```hcl
resource "helm_release" "openwebui" {
  name       = "open-webui"
  repository = "https://helm.openwebui.com/"
  chart      = "open-webui"
  version    = "7.2.0"
  namespace  = kubernetes_namespace.this.metadata[0].name
  atomic     = true

  values = [
    "${file("helm.values.yaml")}"
  ]

  set {
    name  = "image.tag"
    value = "v0.6.33"
  }

  set {
    name  = "serviceAccount.name"
    value = var.kubernetes_service_account_name
  }

  set {
    name  = "serviceAccount.annotations.azure\\.workload\\.identity/client-id"
    value = azurerm_user_assigned_identity.uai.client_id
  }
}
```

### 6. UI 配置

当 Open WebUI 部署完成后，可以按以下步骤配置 Azure OpenAI 连接：

1. 前往 **Admin Panel** → **Connections**
2. 点击 **Add Connection**
3. 选择 **Azure OpenAI** 作为 provider
4. 选择 **Entra ID** 作为 authentication type
5. 填写 Azure OpenAI endpoint 与 deployment 细节
6. 保存连接

## 关键组件说明

### Service Account Annotations

Service account 必须带上如下 annotation，才能与 Azure User Assigned Identity 关联：

```yaml
azure.workload.identity/client-id: <USER_ASSIGNED_IDENTITY_CLIENT_ID>
```

### Pod Labels

Pods 必须包含以下 label，才能启用 workload identity：

```yaml
azure.workload.identity/use: "true"
```

### Federated Identity Credential

Federated identity credential 会在以下三者之间建立信任关系：
- 你的 AKS 集群 OIDC issuer
- Kubernetes service account
- Azure User Assigned Identity

其中 `subject` 字段格式如下：
```
system:serviceaccount:<namespace>:<service-account-name>
```

## 故障排查

### 常见问题

1. **认证失败**
   - 确认 AKS 集群已启用 OIDC issuer
   - 确认 AKS 集群已启用 workload identity
   - 确认 federated identity credential 的 subject 与 namespace / service account name 完全匹配

2. **权限错误**
   - 确认 user-assigned identity 已被赋予 `Cognitive Services OpenAI User` 角色
   - 确认 role assignment 的 scope 覆盖了你的 Azure OpenAI resource

3. **Pod Identity 问题**
   - 确认 pod 带有 `azure.workload.identity/use: "true"` label
   - 确认 service account 带有正确的 `azure.workload.identity/client-id` annotation
   - 确认 service account name 与 federated credential 配置一致

### 验证命令

检查 service account annotations：
```bash
kubectl get serviceaccount <service-account-name> -n <namespace> -o yaml
```

查看 pod labels：
```bash
kubectl get pod <pod-name> -n <namespace> -o yaml
```

检查 workload identity webhook：
```bash
kubectl get mutatingwebhookconfiguration azure-wi-webhook-controller-manager-mutating-webhook-configuration
```
